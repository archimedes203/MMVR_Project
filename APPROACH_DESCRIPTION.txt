3. Setup loss function.

       ---> The loss function for our "fusion" model is described in the comments
            of training\loss.py. It is defined as:
                
                               L = λ_hm * L_heatmap  +  λ_coord * L_coord

               L_heatmap : mean squared error between the model's predicted heatmap and 
                           the ground-truth heatmap
              
              L_coord:    mean absolute error between predicted keypoint coordinates and 
                          ground-truth coordinates 
                     
              λ_hm:       penalty weight for L_heatmap, currently set to 1.0
              λ_coord:    penalty weight for L_coord, currently set to 5.0

              The weights simply tell the model how "important" each aspect is.
              If heatmap accuracy is more important, increase λ_hm. If the exact
              prediction of a joint is more important, increase λ_coord.

       ---> For clarification:
              
              Heatmap: For each of the 17 skeleton keypoints (joints), there is a 2D image where the "hot"
                       region is cenetered on a specific joint's location. This is the zone where the model 
                       is guessing the more general location of a given joint.
              
              Final guess: Happens at the SoftArgmax2D class (models/blocks.py). Heatmap is converted into a 
                           probability distribution. High-confidence regions get more weight, low-confidence 
                           regions less weight. Grid (lines 86-88) creates a grid of (x,y) positions between
                           0 and 1, covering the full heatmap. Then the weighted average (lines 90-91) multiplies 
                           each grid position by the probability (from Sotfmax), producing a single coordinate per 
                           keypoint (this process is done once for the x coordinate and again for the y coordinate)
